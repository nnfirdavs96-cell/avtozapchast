# История изменений — AutoDoc

Документ описывает все доработки, внесённые в рамках сессии полировки и исправления багов.

---

## Слайдер главной страницы

### Независимые настройки для десктопа и мобильного
- Добавлены колонки `text_blocks_mobile` и `text_pos_mobile` в таблицу `sliders`
- В редакторе слайда появились две независимые вкладки **Десктоп** / **Мобильный**
- Каждая версия имеет свои текстовые блоки (размер, шрифт, цвет, жирность, отступ) и своё расположение текста (9 позиций на сетке)
- Кнопка «Скопировать из десктопа» переносит десктоп-настройки в мобильные одним кликом
- Баннер-подсказка показывает, какую версию вы сейчас редактируете

### Управление кнопкой слайда
- Добавлена колонка `button_text` в таблицу `sliders`
- В редакторе — поле «Текст кнопки» (если пусто — используется стандартный текст из локализации)
- Превью в админке обновляет текст кнопки в реальном времени при вводе

### Исправление ссылки кнопки (внутренний IP)
- Кнопка «Магазин» вела на внутренний IP `10.230.13.107` вместо `https://autodoc.tj`
- Добавлена автоматическая миграция: при загрузке `admin/sliders.php` все записи с внутренними IP (`10.x`, `192.168.x`, `172.16-31.x`) исправляются автоматически
- При сохранении слайда внутренние IP всегда обрезаются из `link_url`
- На фронтенде `index.php` тоже нормализует URL перед выводом
- Добавлена подсказка в форме: использовать относительный путь (`/catalog/index.php`)

### Позицирование текста в редакторе (баг)
- Был дублирующийся блок «Расположение текста» — позиция никогда не сохранялась
- Дубль удалён, теперь позиция корректно сохраняется и работает отдельно для каждой версии

### Мобильное изображение — высота и превью
- Высота мобильного слайдера установлена **380 px** (до ≤390 px экранов — 320 px)
- Убрана адаптивная JS-логика, которая давала слишком маленький слайдер для горизонтальных фото
- Превью в админке (мобильный режим): фрейм **390 × 380 px** — точно совпадает с реальным отображением
- Рекомендация изображения обновлена: **~1080 × 1080 px (квадратное)**, важный объект держать в центре

### Рендеринг на фронтенде
- Переработан `index.php`: два абсолютно позиционированных слоя (`.sl-variant--desktop` / `.sl-variant--mobile`)
- На мобильных (< 768 px) скрывается десктоп-слой, показывается мобильный — через CSS `@media`
- Мобильный вариант использует `--fsm` для закрепления размера шрифта без автомасштабирования

## Баннеры

- На мобильной версии — горизонтальный скролл (scroll-snap), баннеры в ряд
- Изображения: `width:100%; height:auto` — показываются полностью, без обрезки

---

## Технические улучшения

- `dbAddColumnIfMissing()` используется для всех новых колонок — безопасно для уже существующих таблиц
- `applyResponsiveBg()` в `app.js` упрощён: только устанавливает фоновое изображение, высота полностью через CSS
- JS-валидация всех изменений через `node --check`, PHP-валидация через `php -l`
- Все изменения через отдельные PR в ветку `main`
